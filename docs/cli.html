Command line reference

Use AnoSecureKit from terminals, scripts, and release checks.

The installed `anosecurekit` executable covers hashes, HMAC, HKDF, random tokens, AES-256-GCM packets, key wrapping, and chunked file sealing.

Read full CLI docs Download releases

Build

Configure with the CLI enabled, build the project, then run the local check target before relying on the executable.

Inputs

Commands accept text, files, strict hex keys, key files, password files, and optional AAD where the format supports it.

Automation

Check the process exit code before trusting output. Treat file-command stdout as binary when using `--out -`.

Quick start

Build from a checkout with tests enabled, then run the default local check surface. 

cmake -S . -B build -DCMAKE_BUILD_TYPE=Release -DANOSECUREKIT_BUILD_TESTS=ON
cmake --build build --config Release --target check

Common commands

anosecurekit --version
anosecurekit token 32
anosecurekit sha256 --text abc
anosecurekit hmac-sha256 --key-hex 4a656665 --text "what do ya want for nothing?"
anosecurekit keygen --out key.hex
anosecurekit seal-file --in plain.bin --out plain.bin.skf --key-file key.hex
anosecurekit verify-file --in plain.bin.skf --key-file key.hex
anosecurekit open-file --in plain.bin.skf --out plain.bin --key-file key.hex

File recipes

# Seal and open a file with a generated key file.
anosecurekit keygen --out key.hex
anosecurekit seal-file --in plain.bin --out plain.bin.skf --key-file key.hex
anosecurekit open-file --in plain.bin.skf --out plain.bin --key-file key.hex

# Bind authenticated context with AAD. Use the same AAD when opening.
anosecurekit seal-file --in plain.bin --out plain.bin.skf --key-file key.hex --aad-text record:v1
anosecurekit open-file --in plain.bin.skf --out plain.bin --key-file key.hex --aad-text record:v1

# Verify without creating a plaintext output file.
anosecurekit verify-file --in plain.bin.skf --key-file key.hex

Behavior to remember

  • Output paths are never overwritten.
  • Wrong keys, wrong passwords, changed AAD, and modified data fail generically.
  • Password files are raw bytes: no trimming, normalization, prompt, or environment lookup.
  • `verify-file` and `verify-file-password` authenticate without writing plaintext.
  • Detailed command shapes live in the full CLI reference.
