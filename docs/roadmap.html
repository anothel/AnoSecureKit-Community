Now

Prepare a v0.4.1 maintenance release from the reviewed fuzz fix and evidence/documentation work. Do not add a new public API or reinterpret SKT1, SKF1, or SKP1 v1.

Verified baseline

v0.4.0 publication evidence is self-contained. Exact OpenSSL/external provider parity passed 124/124 through both assemblies with the same inventory.

Deferred hosted work

Current-main hosted reruns remain DEFERRED_EXTERNAL_BILLING. Local evidence must not be described as hosted PASS.

Roadmap rules

  • docs/NEXT_WORK_QUEUE.md is the only canonical active queue.
  • Keep OpenSSL 3.x as the only shipped Community production provider.
  • Keep the external provider seam build-tree-only with no silent fallback.
  • Do not copy Enterprise proprietary source or an AnoCrypto-C adapter into Community.
  • Do not reinterpret SKT1, SKF1, or SKP1 v1.
  • Completed items belong in release notes, focused status documents, or Git history.

Release-impacting work

Release-impacting changes must pass the configured release preflight:

cmake --build build --config Release --target release-preflight

Before publication, verify archives, checksums, SPDX SBOM, attestations, canonical release notes, and the exact Release-body comparison policy.

Out of scope unless the product changes

  • Novel crypto primitives, TLS, networking, or secure key storage.
  • A second shipped Community provider.
  • Enterprise licensing, activation, customer delivery, or proprietary adapters.
  • KCMVP, FIPS, certification, validation, or security-audit claims without exact evidence.
