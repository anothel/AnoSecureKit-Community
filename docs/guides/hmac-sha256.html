Guide

HMAC-SHA-256 authenticates bytes with a secret key.

Use HMAC-SHA-256 when both sides already share a secret key and need to detect message changes. A plain SHA-256 hash is public and keyless; HMAC is keyed authentication.

What it is

HMAC combines a secret key with SHA-256 to produce a 32-byte tag, commonly printed as 64 lowercase hex characters.

Good uses

Use it for webhook signatures, signed metadata, request verification, and protocols that already define HMAC-SHA-256.

Not encryption

HMAC detects tampering but does not hide data. Use AEAD encryption when confidentiality is required.

When to use it

  • Use HMAC-SHA-256 when the verifier has the same secret key as the signer.
  • Use plain SHA-256 only for public, keyless digests.
  • Use AES-GCM or file sealing when data must be encrypted.
  • Do not invent custom signature formats without a replay and key-rotation plan.

CLI example

The CLI accepts the HMAC key as strict hex and accepts text or file input. Output is lowercase hex. 

anosecurekit hmac-sha256 --key-hex 4a656665 --text "what do ya want for nothing?"
anosecurekit hmac-sha256 --key-hex 4a656665 --file path/to/file.bin

C++ API example

#include "anosecurekit/hash.hpp"
#include "anosecurekit/hex.hpp"

#include <cstddef>
#include <iostream>

anosecurekit::bytes key{std::byte{'J'}, std::byte{'e'}, std::byte{'f'}, std::byte{'e'}};
anosecurekit::bytes message{
    std::byte{'w'}, std::byte{'h'}, std::byte{'a'}, std::byte{'t'}};

anosecurekit::digest256 tag = anosecurekit::hmac_sha256(key, message);
std::cout << anosecurekit::hex_encode(tag) << "\n";

Known vector

5bdcc146bf60754e6a042426089575c75a003f089d2739839dec58b964ec3843

Security notes

  • Keep HMAC keys secret. Anyone with the key can create valid tags.
  • Include all fields that affect meaning in the signed bytes.
  • Use a replay defense when the same signed message should not be accepted twice.
  • Compare expected and received tags with constant-time comparison when values are secret.
  • Rotate keys with protocol support; do not silently change keys without verifier coordination.
