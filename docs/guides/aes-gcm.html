Guide

AES-256-GCM encrypts bytes and authenticates the result.

AnoSecureKit uses AES-256-GCM for `SKT1` packets through its current default OpenSSL backend. The public one-shot API generates nonces internally, authenticates AAD, and returns plaintext only after tag verification succeeds.

What it is

AES-GCM is an authenticated encryption mode. It protects confidentiality and detects tampering when used with unique nonces.

AnoSecureKit shape

`encrypt` returns one `SKT1` packet containing header, random nonce, ciphertext, and authentication tag.

AAD matters

AAD is authenticated but not stored. Decryption requires the exact same AAD bytes supplied during encryption.

When to use it

  • Use AES-GCM when data must stay confidential and tampering must be detected.
  • Use AAD to bind external context such as record type, version, or tenant ID.
  • Use file sealing for files instead of putting large file workflows into one packet.
  • Use HMAC when data is public but must be authenticated with a shared key.

CLI example

Packet commands accept a 32-byte key as 64 hex characters or a key file. With `--out`, the packet is binary and output paths are not overwritten. 

anosecurekit keygen --out key.hex
anosecurekit encrypt --text hello --key-file key.hex --aad-text record:v1 --out hello.skt
anosecurekit decrypt --packet-file hello.skt --key-file key.hex --aad-text record:v1

C++ API example

#include "anosecurekit/aead.hpp"
#include "anosecurekit/random.hpp"

#include <cstddef>

anosecurekit::bytes plaintext{std::byte{'h'}, std::byte{'i'}};
anosecurekit::bytes aad{std::byte{'v'}, std::byte{'1'}};
anosecurekit::key256 key = anosecurekit::generate_key();

anosecurekit::bytes packet = anosecurekit::encrypt(plaintext, key, aad);
anosecurekit::bytes opened = anosecurekit::decrypt(packet, key, aad);

`SKT1` packet shape

  • 5-byte `SKT1` header.
  • 12-byte random nonce generated by AnoSecureKit.
  • AES-256-GCM ciphertext.
  • 16-byte AES-GCM authentication tag.

The packet header and caller-provided AAD are authenticated. AAD is not stored in the packet.

Security notes

  • Callers cannot supply packet nonces in the public one-shot API.
  • Wrong keys, wrong AAD, modified nonce, modified ciphertext, and modified tag fail generically.
  • Do not release plaintext from packet streaming until `finalize(tag)` succeeds.
  • Keep keys out of logs, URLs, command history, crash dumps, and source control.
  • Use a separate security review before using AnoSecureKit in high-risk systems.
