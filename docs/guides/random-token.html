Guide

Random tokens are URL-safe text from random bytes.

AnoSecureKit random tokens are generated from cryptographically secure random bytes and encoded as unpadded Base64URL. The size argument is the number of random bytes before encoding.

What it is

A random token is unpredictable text suitable for identifiers, one-time values, and application-level secrets when stored and handled safely.

Format

AnoSecureKit returns unpadded Base64URL text using `A-Z`, `a-z`, `0-9`, `-`, and `_`.

Storage matters

Token generation is only one part. Applications must protect tokens in logs, URLs, databases, backups, and monitoring systems.

When to use it

  • Use random tokens for session-like identifiers, CSRF-like nonces, invitation codes, and reset handles when the surrounding protocol is sound.
  • Use enough random bytes for the threat model. `32` bytes is a common strong default.
  • Do not use random tokens as encryption keys unless the API specifically expects raw key bytes.
  • Do not regenerate tokens deterministically from user data.

CLI example

The CLI argument is the number of random bytes before Base64URL encoding. `0` is rejected. 

anosecurekit token 32

C++ API example

#include "anosecurekit/random.hpp"

#include <iostream>
#include <string>

std::string token = anosecurekit::random_token(32);
std::cout << token << "\n";

Security notes

  • Generated tokens are secrets if possession grants access.
  • Do not log generated tokens unless logs are protected like secrets.
  • Do not send high-value tokens through referrers, analytics, or third-party scripts.
  • Compare stored token digests or use a keyed design when database disclosure is in scope.
  • Use `anosecurekit::generate_key()` when you need a 32-byte raw key, not text.
