Guide

Base64URL is Base64 made safe for URLs and filenames.

Use Base64URL when encoded values appear in URLs, filenames, or tokens that should avoid `+`, `/`, and `=`. It is still encoding, not confidentiality.

What changes

Compared to standard Base64, Base64URL replaces `+` with `-`, `/` with `_`, and is commonly transmitted without padding.

When to use

Use it for token-like strings, URLs, filenames, and similar contexts where standard Base64 characters are inconvenient or rejected.

What it does not do

It does not encrypt or authenticate. Anyone can decode it, and logs or URLs are still sensitive depending on context.

Input and decoding rules

  • Canonical Base64URL input is stricter than symbol replacement alone.
  • Padding (`=`) is not used for the canonical output form.
  • Malformed length or character set is rejected.

CLI example

AnoSecureKit provides dedicated commands for URL-safe encode/decode.

anosecurekit base64url-encode --text abc
anosecurekit base64url-decode --text YWJj

C++ API example

#include "anosecurekit/base64.hpp"

#include <cstddef>
#include <string>

anosecurekit::bytes data{std::byte{'f'}, std::byte{'o'}, std::byte{'o'}};
std::string token = anosecurekit::base64url_encode(data); // "Zm9v"
anosecurekit::bytes decoded = anosecurekit::base64url_decode(token);
