Guide

SHA-256 gives bytes a fixed 32-byte digest.

Use SHA-256 to identify data, compare known content, or build higher level protocols that already require it. Do not use a plain hash as encryption, authentication, or password storage.

What it is

SHA-256 is a cryptographic hash function. It maps any byte input to a 32-byte digest, usually printed as 64 lowercase hex characters.

Good uses

Use it for checksums, content identifiers, test vectors, and protocols that already specify SHA-256.

Bad uses

A plain hash is not encryption, not a MAC, and not password hashing. Use HMAC, AEAD, or a password KDF where appropriate.

When to use it

  • Use SHA-256 to verify that public data matches an expected digest.
  • Use it to name immutable content when collision resistance is enough for the protocol.
  • Use HMAC-SHA-256 when a secret key must authenticate data.
  • Use encryption when data must stay confidential.

CLI example

The CLI prints SHA-256 digests as lowercase hex. It accepts text or a file path. 

anosecurekit sha256 --text abc
anosecurekit sha256 --file path/to/file.bin

C++ API example

#include "anosecurekit/hash.hpp"
#include "anosecurekit/hex.hpp"

#include <cstddef>
#include <iostream>

anosecurekit::bytes data{std::byte{'a'}, std::byte{'b'}, std::byte{'c'}};
anosecurekit::digest256 digest = anosecurekit::sha256(data);

std::cout << anosecurekit::hex_encode(digest) << "\n";

Known vector

`sha256("abc")` is:

ba7816bf8f01cfea414140de5dae2223b00361a396177a9cb410ff61f20015ad

Security notes

  • SHA-256 is deterministic: the same input always gives the same digest.
  • Anyone with the input can recompute the digest.
  • Use `hmac_sha256` or `anosecurekit hmac-sha256` when a secret key must protect integrity.
  • Use `seal-file`, `encrypt`, or the C++ AEAD APIs when you need confidentiality.
  • Do not store account passwords as raw SHA-256 digests.
