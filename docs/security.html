Security boundaries

Know what AnoSecureKit protects, and what it does not.

AnoSecureKit is a small C++20 utility library with OpenSSL 3.x as the current default crypto backend. It has not had an external security audit and is not a substitute for a full review in high-risk systems.

Read security model Report policy

Scope

AnoSecureKit provides byte encoders, SHA-256, HMAC, HKDF, random bytes, AES-256-GCM packets, and chunked file sealing through a private crypto backend boundary. OpenSSL is the current default backend.

Audit status

AnoSecureKit has not had an external security audit. Use separate review for high-risk systems.

Reports

Do not report vulnerabilities through public issues. Use GitHub Security Advisories or a private maintainer contact path.

What AnoSecureKit does not provide

  • Custom cryptographic primitives.
  • TLS, networking, or secure key storage.
  • Guaranteed memory erasure.
  • Password hashing for account authentication.
  • Caller-selected algorithms or nonces.
  • Protection from compromised hosts, filesystems, shells, or build tools.

Authentication rules

One-shot decrypt returns plaintext only after authentication succeeds. Incremental `packet_decryptor::update()` can return unverified plaintext before `finalize(tag)`, so callers must buffer it until final authentication succeeds.

Wrong keys, wrong AAD, wrong passwords, modified ciphertext, modified nonces, and modified tags use generic authentication failures.

File output safety

Path-based open commands refuse to overwrite output paths and commit a temporary file only after successful authentication. Stream output and CLI `--out -` cannot be rolled back by AnoSecureKit, so callers must trust output only after the command exits successfully.

Password-based file sealing

`SKP1` reads password bytes exactly as supplied. AnoSecureKit does not trim, normalize, prompt, read environment variables, or define password quality rules. Current `SKP1` uses fixed scrypt parameters.

OpenSSL dependency

AnoSecureKit uses the default OpenSSL library context and provider configuration already active in the process. Applications that require custom providers must configure OpenSSL before calling AnoSecureKit.
