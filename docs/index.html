OpenSSL-backed today, backend-neutral by design

Practical C++20 crypto utilities.

AnoSecureKit gives application code a compact byte-oriented API for encoding, hashing, random tokens, AES-256-GCM packets, key wrapping, and chunked file sealing without spreading provider calls everywhere.

View on GitHub Use the CLI

What it is

A small C++20 library for common binary and cryptographic tasks in application code.

What it covers

  • Hex, Base64, and Base64URL
  • SHA-256, HMAC-SHA-256, and HKDF-SHA-256
  • Constant-time comparison for equal-length values
  • Random bytes, random keys, and URL-safe tokens
  • AES-256-GCM packets, key wrapping, and file sealing

Security status

No external security audit yet. Do not use it as a replacement for a full review in high-risk systems.

Quick start

Build the project and run the default local checks from a source checkout. 

cmake -S . -B build -DCMAKE_BUILD_TYPE=Release -DANOSECUREKIT_BUILD_TESTS=ON
cmake --build build --config Release --target check

Web access plan

This site starts as a documentation-first project page with small browser-local tools. Plaintext, keys, passwords, and files should not be uploaded to a server for routine crypto operations.

Any future web tools should run locally in the browser, avoid third-party scripts on tool pages, and clearly separate documentation or advertising areas from security-sensitive input areas.
